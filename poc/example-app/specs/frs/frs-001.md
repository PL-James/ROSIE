# JWT Token Authentication

## Functional Specification

The system shall implement JSON Web Token (JWT) based authentication for secure user sessions.

## Requirements

### FR-001.1 Token Generation
- Generate JWT tokens upon successful credential validation
- Include user identifier and role claims in token payload
- Sign tokens using HS256 algorithm with secure secret key

### FR-001.2 Token Validation
- Validate token signature on each authenticated request
- Verify token expiration timestamp
- Extract and validate user claims

### FR-001.3 Token Expiration
- Tokens shall expire after 24 hours by default
- Expiration time shall be configurable via environment variable
- Expired tokens must be rejected with appropriate error response

## Interface Specification

```typescript
interface JWTPayload {
  sub: string;      // User identifier
  role: string;     // User role
  iat: number;      // Issued at timestamp
  exp: number;      // Expiration timestamp
}

function generateToken(userId: string, role: string): string;
function validateToken(token: string): JWTPayload | null;
```
